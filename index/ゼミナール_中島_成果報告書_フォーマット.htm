1.    はじめに

（１）File APIとは

 

l  File APIとは、OSのファイルシステム上にあるファイルをJavaScriptを使い読み取るためのAPIである。

l  スクリプトから自由にファイルシステム上のファイルを指定して読み込む事は出来ない（セキュリティの観点から）

l  扱えるファイルは、Type属性に“file”が指定されたinput要素からユーザーが選択したファイル、または、ユーザーが所定のウェブコンテンツ領域にドロップしたファイルに限られる。

l  ファイル名やファイルサイズだけでなく、ファイルの中身、テキストのみならず、バイナリ―ファイルも読み取れる。

2.   File Reader

（１） File Readerとは

『サンプルデータ０１』のようにファイルシステム上の画像ファイル(オレンジの枠で囲んだ部分)をマウスを使ってオレンジ矢印の場所(グレーの四角い範囲はdiv要素)にドロップすると

 

 

『サンプルデータ０１』

 

 

 

　　　『サンプルデータ０２』のようにそれをimg要素として生成してdiv要素に追加する。

 

 

 

 

『サンプルデータ０２』

 

　File APIの重要なオブジェクトは、dropイベントのリスナー関数で取り出したFileListオブジェクトと、その中に格納されているFileオブジェクトである。

　ドロップされたファイルをリストとして取り出せる理由は、同時に複数のファイルをまとめてドロップ出来るからである。

　Fileオブジェクトは、ドロップされたファイルを表しており、このオブジェクトを通して、さまざまなファイルの情報を読み取ることができる。

　Fileオブジェクトのさまざまなプロパティを使って、いろいろな種類のファイルの情報を取り出すことができる。

　また、Fileオブジェクトは、Blobオブジェクトを継承している。そのため、下の表には、Blobオブジェクトのプロパティも合わせて表示する。

 

 

表：【 Blob / Fileオブジェクトのプロパティ】

プロパティ

説明

size

ファイルのサイズを返す。単位はバイト

type

ファイルの種類を表すMIMEタイプを返す。

name

ファイル名を返す。

lastModifiedDate

最終更新日時を表すDateオブジェクトを返す。

 

 

 

　実は、Fileオブジェクトには、ファイルの中身を読み取るメソッドやプロパティはない。よって、FileReaderオブジェクトを通して、ファイルの中身を読み取っている。

 

コンストラクタからインスタンスを作成する。

var reader = new FileReader();

 

 

 

(2)　FileReaderの読み込みメソッド

FileReaderには非同期な3つの読み込みメソッドが定義されている、いずれかを使いファイルを読み込む。

表：【 FileReaderオブジェクトのファイル読み取りメソッド 】

引数

説明

ファイルの中身をArrayBufferで読み取る。

ファイルの中身をテキストで読み取る。

また、W3Cの仕様書には記載されていない読み込みメソッドも利用できる

メソッド

引数

説明

readAsBinaryString

BlobかFile

ファイルをバイナリ形式で読み取る。

 

　　　　これらのファイル読み取りメソッドは非同期のため、FileReaderオブジェクトのonloadプロパティに、ファイル読み取りが完了した時に呼び出されるコールバック関数を指定する。

　　　　読み取りが完了すると、FileReaderオブジェクト(変数reader)のresultプロパティから、呼び出したファイル読み取りメソッドに合わせた形式で、ファイルのデータを取得することができる。

 

 

（３） Blobオブジェクト

 

Blobオブジェクト、実はこれが本当の意味でのファイルのデータを表しているオブジェクトといってもよい。なぜなら、必ずしもファイルのデータは、ファイルという実態を伴うとは限らないからだ。ネットワークを通して取得した画像データは、データそのものにファイル名や最終更新日時というものをもっているわけではない。ファイルという形態に保存されて初めてファイル名などの情報が結び付けられる。このBlobオブジェクトは、さまざまなAPIでも利用されており、近年のウェブAPIでは非常に重要なオブジェクトである。

 

 

 

 

3.   Blob URL

　（１）Blob URLとは

先のサンプルデータのようにファイルをドロップして０２の状態になるまで、実は無駄なデータ変換を繰り返している。せっかくバイナリ―として画像を読み込んだのだから、そのままデコードして表示したい。Blobオブジェクトは、仮想的なURLを持たせることができる。
